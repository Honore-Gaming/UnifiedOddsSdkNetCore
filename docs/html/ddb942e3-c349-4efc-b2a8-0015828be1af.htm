Uof Sdk object

UofSdk is the root object of the SDK library. Through it the ConfigurationBuilder and SessionBuilder are accessible.

When everything is configured, the rabbit connection is established after calling uofSdk.Open() and then you start receiving the feed messages. Open/Close methods are meant to be called just once (at the beginning/end of processing). If you need to call the Open method again, a new uofSdk instance should be created.

All the feed messages will come in UofSession event handlers.

Global events ( such as feed disconnections, producer up/down, producer status changes, ...) can be handled in UofSdk event handlers.

For special scenarios, two dedicated UofSdk extended classes are available. UofSdkForReplay is meant for connecting to the replay server and UofSdkExtended when there is a need to receive raw feed and/or API data.

Managers/Providers

UofSdk instance also provides access to a variety of managers/providers for additional actions and functionality.

IProducerManager
Can be used to get insight into available producers, when the recoveries were done, associated recovery request id, etc.

ISportDataProvider
Exposes methods for fetching sport-related data (sports, tournaments, seasons, competitions, lotteries, etc), deleting cache items, exporting/importing cache items, etc. Most of the methods will invoke API requests.

IMarketDescriptionManager
Can be used for getting information about available markets, translations for markets, and outcomes (including outrights).

ICashOutProbabilitiesProvider
Can be used for providing information about cashout probabilities for betting markets.

IBookingManager
Can be used to book new sport events.

ICustomBetManager
Can be used to perform various custombet operations.

IEventChangeManager
Can be used to listen for result and fixture changes. Can be also setup to get periodical new changes automatically.

IEventRecoveryRequestIssuer
Can be used to issue recovery requests for specific event to the feed.
